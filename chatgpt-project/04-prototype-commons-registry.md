# Prototype Commons Registry

The registry is a normalized view of informal artifacts. It is not an approval list.

## Registry columns

| Column | Meaning |
|---|---|
| ID | Short identifier |
| Artifact | Name |
| Type | Script, GPT, agent, dashboard, automation, app, workflow |
| Owner | Primary owner |
| Backup | Backup owner |
| Users | Scope of use |
| Current class | Provisional production class |
| Reliance level | Low, moderate, high, unknown |
| Data sensitivity | None, low, internal, confidential, regulated, customer, unknown |
| Systems touched | None, read-only, write-enabled, system of record, external, unknown |
| Customer exposure | None, indirect, direct, unknown |
| Value signal | Weak, anecdotal, directional, strong, unknown |
| Support exposure | Low, moderate, high, unknown |
| Route | Keep personal, hold beta, promote, formalize, demote, retire |
| Open questions | Missing evidence needed |

## Registry rules

- Do not overstate certainty.
- Use provisional class until evidence is sufficient.
- Capture why an artifact matters and what it touches.
- Highlight unmanaged business reliance.
- Flag artifacts that need immediate owner clarification.
