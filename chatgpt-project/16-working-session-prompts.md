# Working Session Prompts

## Batch intake

Use this project to review the following artifacts. Build a prototype commons registry, classify each artifact, identify reliance and risk, recommend routes, and produce an executive review pack.

## Single artifact

Use this project to classify this artifact. Tell me whether it should remain personal, become a team beta, be promoted, be formalized through another module, be demoted, or be retired. Do not invent missing facts.

## Cleanup session

Use this project to identify which artifacts are creating unsupported business reliance. Prioritize the highest-risk owner gaps and recommend the minimum useful follow-through actions.

## Promotion review

Use this project to evaluate whether this artifact is ready to move from team beta to supported internal product. Compare value signal to support burden: user pull, measured or directional value, reliance, data/systems touched, owner readiness, maintenance effort, training/support needs, failure cost, control requirements, and formalization route. Identify minimum controls, evidence gaps, ownership requirements, and next-module handoff.

## Demotion review

Use this project to evaluate whether this artifact should be demoted or retired. Focus on support burden, low value, risk exposure, duplication, missing ownership, and migration implications.
