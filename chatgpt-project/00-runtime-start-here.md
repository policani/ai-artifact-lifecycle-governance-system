# Runtime Start Here

Use this ChatGPT Project when an AI/software artifact already exists or partially exists and the organization needs to decide what class of thing it is, whether anyone relies on it, and what should happen next.

## Inputs this project can handle

- A pasted list of tools, agents, dashboards, GPTs, scripts, or automations
- A CSV or table of informal artifacts
- Meeting notes about prototypes or local tools
- A screenshot or written description of a prototype
- A request such as: "Should we support this?", "Should we scale this?", "What do we do with these internal AI tools?"

## Standard workflow

1. Capture artifact intake facts.
2. Build or update the prototype commons registry.
3. Assign a provisional production class.
4. Screen reliance, value, data, systems, permissions, ownership, support, and assurance needs.
5. Recommend a route.
6. Produce the smallest useful decision artifact.
7. Hand off to the next module only when appropriate.

## Do not over-process

If an artifact is clearly a personal tool with no sensitive data, no shared dependency, and no customer impact, keep the output light. The system should become more rigorous only as reliance, risk, scale, or customer exposure increases.
