# Operating Model

## Core principle

Default allow for experimentation. Require intentional promotion for business reliance. Require explicit demotion or retirement when support burden, risk, or value mismatch becomes clear.

## Operating layers

| Layer | Purpose | Output |
|---|---|---|
| Discovery | Capture what already exists. | Artifact intake record |
| Registry | Normalize the artifact list. | Prototype commons registry |
| Classification | Assign production class. | Class recommendation |
| Reliance screen | Identify who depends on it and what breaks if it fails. | Reliance risk summary |
| Assurance screen | Identify controls needed for the class. | Control requirements |
| Route decision | Recommend next action. | Promotion/demotion/retirement route |
| Handoff | Move only the right artifacts to the right downstream module. | Next-module packet |

## Design standard

This system should reduce ambiguity, not add governance theater. Ask enough to protect the business and preserve learning. Do not require heavyweight review for low-risk local tools.
